What is Agentic AI?

Chat-based AI (e.g., ChatGPT, Claude.ai)

  • You ask a question, you get an answer
  • One turn at a time, copy-paste workflow

Agentic AI (e.g., Claude Code, OpenAI Codex, Gemini CLI, GitHub Copilot)

  • Plans multi-step tasks autonomously
  • Uses tools: reads/writes files, runs code, searches the web
  • Iterates on errors, tests its own work
  • Works directly in your project

Key shift: from answering questions to doing work.

Prerequisites: Git & GitHub

Version control is essential when working with AI agents.

  • AI agents make changes to your files directly
  • You need to review what the agent did
  • You need to revert changes that don’t work
  • Git gives you a safety net and an audit trail

Optional Prerequisites: The Terminal

Many cutting-edge agentic tools are terminal-first:

  • Claude Code, OpenAI Codex, Gemini CLI, OpenCode
  • Most also have VS Code/IDE extensions (GitHub Copilot, Cursor)

Getting comfortable with the terminal matters because:

  • More control over agent behavior
  • Direct access to logs and tool output
  • Easier to integrate into reproducible workflows
  • Easier to monitor and debug agent actions

Building Blocks of AI Agents

Project context files

  • CLAUDE.md / AGENTS.md — tell the agent about your project, coding standards, and workflows

Tool use

  • File read/write, bash commands, web search, API calls

Model Context Protocol (MCP)

  • Extensible protocol for connecting agents to external tools (databases, Notion, Slack, etc.)

Skills & slash commands

  • Reusable prompts and instructions for common tasks (e.g., /commit, /review-pr)

Sub-agents

  • Specialized workers for parallel or focused tasks (e.g., code review, testing, exploration)

Context is King

Context is key to getting good results from AI agents.

  • Put everything related to the project in the same repository so the agent has a full overview
  • Be precise about what you want and explain why — this yields better results
  • Agents and sub-agents help minimize context usage by focusing on specific tasks

Watch out for context limits:

  • Long sessions lead to compaction — the agent summarizes older messages and may forget relevant details
  • Compaction can degrade output quality as important context is lost
  • Start fresh sessions for new tasks when context gets heavy

Properly managing context is a skill you develop as you use the system.

Security: Don’t Trust

AI agents can execute code and modify files. Risks include:

  • Prompt injection — malicious instructions hidden in data
  • Data exfiltration — agent sends sensitive data to external services
  • Destructive actions — deleting files, overwriting work, force-pushing
  • Incorrect code — agent produces code that runs but doesn’t do what you expect, leading to wrong research results

Mitigation strategies:

  • Use permission controls — review before approving tool calls
  • Use containerization — dev containers, Docker, VMs
  • Use Git — commit before letting the agent work, review diffs after
  • Never give agents access to sensitive information

Verify: Ensuring Code Accuracy

AI-generated code may run without errors but produce wrong results. You must verify.

Unit tests

  • Write tests that check expected outputs for known inputs
  • Agents can generate tests — but review them carefully

Defined schemas & documentation

  • Document what each processing step should do
  • Use a second agent to validate that code matches the documentation

Sniff tests & descriptive statistics

  • Generate sanity checks: are means reasonable? Do signs make sense?
  • Look at intermediate outputs to catch errors early
  • Build helper functions that summarize results in human-readable ways, just like in pre-AI times

Validating that code does what you expect is not optional — it’s essential.

Best Practices for Agentic Research

  1. Start with a CLAUDE.md / AGENTS.md — give the agent project context, coding standards, and data descriptions. Update it regularly.

  2. Plan before doing — have the agent create a plan, then review it before implementation begins.

  3. Give the agent what it needs — ensure access to all relevant data, documentation, and context.

  4. Generate validation data — create intermediate outputs, simulated data, or known-answer datasets to check against.

  5. Write tests, then review them — have the agent write unit tests, but always verify the tests themselves are correct.

  6. Build helper functions for inspection — figures, tables, summary stats — anything that makes outputs easy to validate visually.

  7. Use a second agent for code review — a fresh agent can catch mistakes the first one missed.

  8. Document everything — have the agent produce clear documentation of its processing steps, making it easier for you and other agents to review.
