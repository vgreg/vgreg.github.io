2  Installing Python

In this Chapter, we install Python and the tools we need for this book. We cover two ways to install Python: locally and in the cloud. I recommend installing Python locally, but if you are not sure if you want to install Python on your computer, you can use Python in the cloud.

Python in the cloud

Not sure if you want to install Python on your computer? No worries, you can use Python in the cloud. I recommend GitHub Codespaces, which allows to run an environment identical to the one we use in this course directly in your browser. You can find the instructions here.

2.1 Local installation

The most common way to use Python is to install it on your computer. This is called a local installation. The advantage of a local installation is that you can use Python even if you don’t have an internet connection. The disadvantage is that you need to install Python and the tools we use on your computer. This can be a bit tricky, especially if you are new to programming. The instructions below will guide you through the process of installing the following tools:

  • Python: The Python interpreter, which allows you to run Python code.
  • Poetry: A package manager for Python. We use it to manage the external libraries we use in our projects.
  • Visual Studio Code: A code editor that we use to write Python code.
  • Git and GitHub: We use Git to manage our code and GitHub to host our code online and collaborate with others.
  • The instructions will provide you with a working Python environment that you can use to follow along with the examples in the next chapter. I cover these tools in more detail in later chapters.

2.1.2 Poetry

  • Mac
  • Linux
  • Windows

Most Python projects use external libraries. For example, we use the pandas library for data analysis. To manage these libraries, we need a package manager. I recommend using Poetry.

Installation instructions can be found here.

By default, Poetry installs Python for each project in the ~/Library/Caches/pypoetry/virtualenvs/ directory. I prefer to have it in the project directory, that way if, I delete the directory, then the environmnent is deleted as well. To enable this, run the following command in the Terminal app:

poetry config virtualenvs.in-project true

2.1.4 Git and GitHub

  • Mac
  • Linux
  • Windows

We rely on Git and GitHub to manage our code. Git is a version control system that allows us to track changes to our code. GitHub is a website that hosts Git repositories. Git is already installed on Mac OS as a command-line tool. You can also use Git directly in VS Code, or using a GUI client such as GitHub Desktop. I prefer to use the VS Code integration or the command-line tool, but many beginners prefer to use GitHub Desktop.

To follow along the coming chapters, you will need to create a GitHub account. You can create a free account at https://github.com/.

GitHub offers many benefits to students and educator, including a free access to GitHub Copilot and extra free hours for GitHub Codespaces. I highly recommend applying at GitHub Education if you are eligible.

2.2 Python in the cloud using Github Codespaces

Many online platforms allow you to develop and run Python code without installing anything on your computer. If you want to use a cloud-based solution, I recommend using GitHub Codespaces.

All you need is a GitHub account. However, note that GitHub Codespaces is not free. At the time of this writing, you get 60 hours per month for free, or 90 hours if you signed up for the GitHub Student Developer Pack (this is for a 2-core machine, which is the smallest machine available). After that, you have to pay for it (the current rate is USD 0.18 per hour).

Make sure to shut down your Codespace when you are not using it, otherwise you will run out of free hours very quickly.

Setting up a new Codespace is not hard, but it is a bit long, so we come back to it in a later chapter. For now, I have prepared a template repository to get started and run the code in the next chapter.
